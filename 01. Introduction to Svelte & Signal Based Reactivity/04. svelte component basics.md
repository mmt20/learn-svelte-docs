# Svelte Component Basics

A Svelte component is a standard `.svelte` file composed of three main parts:
1.  **Script:** Logic (JavaScript/TypeScript)
2.  **Template:** Structure (HTML)
3.  **Style:** Presentation (CSS)

All three are optional, but usually, you'll use a mix of them.

## 1. The Structure

```html
<script>
  // Logical part: Imports, State, Effects
  let name = 'Svelte';
</script>

<!-- Template part: The markup -->
<h1>Hello, {name}!</h1>

<style>
  /* Style part: Scoped CSS */
  h1 {
    color: #ff3e00;
  }
</style>
```

## 2. Displaying Data

To display data, wrap a variable in curly braces `{}`. This works for attributes too.

```html
<script>
  let src = 'logo.png';
  let name = 'Logo';
</script>

<img {src} alt={name} />
```
*Note: `{src}` is shorthand for `src={src}` when the attribute name matches the variable name.*

## 3. Component Props (Inputs)

In Svelte 5, we use the `$props` rune to accept data from a parent component.

### Child.svelte
```html
<script>
  // Destructure props simply
  let { title, count = 0 } = $props(); 
</script>

<p>{title} has count: {count}</p>
```

### Parent.svelte
```html
<script>
  import Child from './Child.svelte';
</script>

<Child title="Counter Item" count={5} />
<!-- fallback default count used here -->
<Child title="New Item" /> 
```

## 4. Scoped Styling

Styles in a `.svelte` file are **scoped by default**. This means if you style a `p` tag in one component, it *will not* affect `p` tags in other components.

```html
<!-- Card.svelte -->
<p>I am red inside this card.</p>

<style>
  p { color: red; } /* Only affects p tags in this file */
</style>
```

## 5. Reactivity with Runes

In Svelte 5, reactivity is opted-in using **RUNES**. The most fundamental rune is `$state`.

*   **Normal variables** (`let x = 1`) are **non-reactive**. Changing them won't update the UI.
*   **State variables** (`let x = $state(1)`) are **reactive**. Changing them triggers granular updates in the DOM.

### Adding Reactivity
To make a component interactive, define state variables and update them in event handlers.

```html
<script>
  let count = $state(0);
  let name = $state('Svelte');

  function increment() {
    count += 1;
  }
</script>

<button onclick={increment}>
  Clicks: {count}
</button>

<input bind:value={name} />
<p>Hello, {name}!</p>
```

*Note: Svelte's reactivity is fine-grained. Only the parts of the DOM that depend on `count` or `name` will update when they change.*

## Summary
*   **`.svelte` files** contain script, html, and style.
*   **`{}`** syntax is used for interpolation.
*   **`$props()`** is used to declare inputs.
*   **CSS is scoped** automatically.
